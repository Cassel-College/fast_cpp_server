12// =============================================================================
13// MyFlyControlManager — 飞控管理器（单例包装层）
14//
15// 设计目的：
16// 1. 为上层业务提供全局唯一入口，避免在业务代码中到处传递 MyFlyControl 对象
17// 2. 保留 MyFlyControl 自身的可实例化能力，不把底层通信对象强行改造成单例
18// 3. 提供统一的生命周期管理：Init / Start / Stop / Shutdown
19// 4. 在 Shutdown 时重建底层对象，清空旧回调和旧状态，方便重初始化与测试
20//
21// 使用建议：
22// - 应用层优先使用 MyFlyControlManager::GetInstance()
23// - 若未来存在多飞控、多串口、多实例需求，直接继续使用 MyFlyControl 即可
24// =============================================================================
